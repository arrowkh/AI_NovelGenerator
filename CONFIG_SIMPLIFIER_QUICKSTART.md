# 配置简化器快速入门指南

## 🚀 5 分钟快速上手

### 1. 运行演示程序

```bash
# 确保已安装依赖
pip install customtkinter

# 运行演示程序
python config_simplifier_demo.py
```

### 2. 选择您的模式

#### 如果您是新手
✅ 点击 **[基础]** 按钮
- 选择 "平衡(推荐)" 预设
- 调整温度滑块（可选）
- 点击 "保存配置"
- 完成！

#### 如果您有经验
✅ 点击 **[高级]** 按钮
- 配置 API Key 和模型
- 调整生成参数
- 启用优化选项
- 点击 "测试配置" 验证
- 保存

#### 如果您是专家
✅ 点击 **[专家]** 按钮
- 查看所有配置项
- 导入已有配置
- 进行高级验证
- 导出分享

### 3. 查看历史记录

```
点击 "📜 查看历史" 按钮
→ 查看所有配置变更
→ 点击 "恢复到此版本" 回退
→ 查看影响和满意度
```

### 4. 查看使用统计

```
点击 "📊 使用统计" 按钮
→ 查看最常用的预设
→ 查看满意度分布
→ 优化配置策略
```

## 📖 核心概念

### 预设配置

| 预设 | 适合场景 | 成本 | 速度 | 质量 |
|------|----------|------|------|------|
| 快速 | 快速原型、初稿 | 💰 低 | ⚡ 快 | ⭐⭐⭐ |
| 平衡 | 日常写作 | 💰💰 中 | ⚡⚡ 中 | ⭐⭐⭐⭐ |
| 高质量 | 重要章节、出版 | 💰💰💰 高 | ⚡ 慢 | ⭐⭐⭐⭐⭐ |
| 创意 | 奇幻、科幻 | 💰💰 中 | ⚡⚡ 中 | ⭐⭐⭐⭐ |

### 温度参数

```
0.0 - 0.3    严谨、一致   适合: 技术文档、历史
0.4 - 0.7    平衡         适合: 现代小说、传记
0.8 - 1.0    创意、想象   适合: 科幻、奇幻
1.1+         实验性       适合: 前卫创作
```

### 配置模式

```
基础模式:  2-3 个选项   → 5 分钟上手
高级模式:  10-15 个选项 → 1 周掌握
专家模式:  30+ 个选项   → 1 月精通
```

## ⚠️ 常见问题

### 问: 我应该选择哪个模式？

**答**: 
- 第一次使用？→ 基础模式
- 用了 1-2 周？→ 高级模式  
- 想完全控制？→ 专家模式

### 问: 温度设置多少合适？

**答**:
- 不确定？→ 0.7（默认）
- 需要准确？→ 0.5-0.6
- 需要创意？→ 0.8-0.9

### 问: 为什么生成质量不好？

**答**: 检查这些：
1. 温度是否过高（>1.0）？
2. Max Tokens 是否过低（<4096）？
3. 是否启用了一致性检查？
4. 尝试切换到 "高质量" 预设

### 问: 如何降低成本？

**答**:
1. 使用 "快速" 预设
2. 降低 Max Tokens
3. 选择更便宜的模型（如 GPT-3.5）
4. 禁用不必要的优化选项

### 问: 生成太慢怎么办？

**答**:
1. 启用 "并行生成"（高级模式）
2. 降低 Max Tokens
3. 选择更快的模型
4. 检查网络连接

### 问: 历史记录会一直保存吗？

**答**: 系统保留最近 50 条记录，足够日常使用。

### 问: 可以导入别人的配置吗？

**答**: 可以！在专家模式点击 "导入"，选择配置文件即可。

### 问: 配置会影响所有项目吗？

**答**: 配置是全局的，但您可以：
- 为不同项目导出不同配置
- 在项目间快速切换配置
- 使用历史记录快速恢复

## 🔧 故障排除

### 无法保存配置

**可能原因**:
- 配置文件权限问题
- 磁盘空间不足
- 配置格式错误

**解决方法**:
1. 检查文件权限
2. 检查磁盘空间
3. 使用 "验证" 功能检查配置

### 验证失败

**可能原因**:
- API Key 未设置
- 参数超出范围
- 配置冲突

**解决方法**:
1. 查看验证错误详情
2. 使用自动修复
3. 参考推荐值调整

### 测试配置失败

**可能原因**:
- API Key 无效
- 网络连接问题
- API 服务不可用

**解决方法**:
1. 检查 API Key 是否正确
2. 检查网络连接
3. 检查 Base URL 是否正确
4. 查看错误日志

### 历史记录丢失

**可能原因**:
- 历史文件被删除
- 文件损坏

**解决方法**:
- 历史记录存储在 `config_history.json`
- 检查文件是否存在
- 如果文件损坏，系统会自动重建
